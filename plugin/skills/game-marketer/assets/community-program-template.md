# Community program — <game title>

> Scaffolds for the community programs running in parallel: Discord, Reddit, creator / influencer, press, ambassador.

## Discord

### Server structure

- **Welcome / rules** — pinned; read-only; auto-roles on entry
- **Announcements** — dev-only; mirrors official channels
- **General chat** — community discussion
- **Looking for group** (multiplayer games)
- **Bug reports** — funnel to support / engineering
- **Suggestions** — read-only feedback inbox; players upvote
- **Fan content** — art, screenshots, clips
- **Off-topic** — relief valve; keeps general clean

### Engagement cadence

- **Daily:** moderator presence, basic Q&A
- **Weekly:** dev-in-channel time (set hours; predictable)
- **Monthly:** community event (giveaway, screenshot contest, community vote)
- **Per-launch / per-season:** launch parties, AMA sessions

### Moderation

- 24/7 coverage in major timezones
- Clear rules; transparent enforcement
- Whitelist of moderators; rotated
- Automated tooling for raids / spam / NSFW

### KPIs

- Member count growth
- Daily active members
- Reaction / engagement rate
- Sentiment (positive / neutral / negative ratio)

## Reddit

### Subreddit (if exists)

- **Officially affiliated** subreddit (don't astroturf)
- **Community-led** moderation; team contributes but doesn't dominate
- **Weekly threads:** new player Q&A, screenshot Saturday, balance discussion
- **Patch threads:** team responds within 24h to top concerns

### Cross-posting

- **Dev posts** for major announcements (launches, content drops, comms)
- **AMAs** at launches and seasonal events
- **Restraint:** don't flood the sub with marketing

## Creator / influencer program

### Tier-1 (top creators in the genre)

- Direct outreach with custom keys / early access
- Briefing call with developers
- Custom assets / demo builds for early coverage
- Long-term relationship (multiple launches, ongoing content)

### Tier-2 (mid-creator pool)

- Code drops via creator network
- Self-serve assets (press kit + creator kit)
- Community manager point of contact

### Tier-3 (long-tail / community creators)

- Ambassador program: free in-game cosmetics for X views
- Affiliate codes (where appropriate)
- Self-serve via website creator portal

### KPIs

- Earned media impressions
- Sentiment of coverage
- Install attribution from creator codes
- Content reuse (creators making multiple videos / streams)

## Press / journalist program

- **Press kit** with high-res assets, fact sheet, dev quotes, demo build
- **Embargo strategy** for major launches (review keys go out 1–2 weeks early)
- **Direct relationships** with key outlets (PC Gamer, IGN, Eurogamer, RPS, Kotaku — depends on category)
- **Indie-specialist outlets** for indie games
- **Mobile-specialist outlets** for mobile games
- **Web3-specialist outlets** for web3 games
- **Press at events** (PAX, Gamescom, Tokyo Game Show, GDC, etc.) — specific outreach windows

## Ambassador / superfan program

For dedicated long-term players:

- Beta access to new content
- Direct feedback channel to devs
- Community moderation training
- Custom in-game recognition (titles, badges)
- Annual gift / shoutout

A small, well-loved ambassador group is worth more than a large transactional creator pool.

## Localized communities

For non-English markets:

- Local Discord server / channel (or section within main server)
- Local moderator / community manager (native speaker)
- Localized announcements (don't just translate; adapt tone)
- Local creator / influencer outreach

Especially important for: JP, KR, ZH, BR, MX, TR. Western-only English communities miss large audiences.

## Web3-specific community notes

If the rails include web3:

- **Holders' channel** in Discord — gated by wallet ownership; inner circle for NFT / token holders
- **Treasury / governance comms** — if community votes on roadmap or treasury, transparent comms required
- **Secondary market integration** — direct players to verified marketplaces; warn against scams
- **Anti-scam vigilance** — phishing in DMs, fake mint links, impersonation accounts; community manager priority

## Crisis comms

When something goes wrong (bug, monetization issue, regulatory issue, exploit, leak):

1. **Acknowledge fast** — silence amplifies the problem
2. **State the facts** — what happened, what's known, what's not yet known
3. **Commit to action** — what the team is doing, when an update is expected
4. **Update on schedule** — even if "still investigating"
5. **Resolve and learn** — postmortem published when appropriate

Coordinate with `game-monetization-strategist` if monetization is involved, `iap-manager` if catalog issue, `security-engineering` if exploit / fraud.

## Hand-off

- **`game-marketer` (this skill):** owns the program structure
- **Community managers (CMs):** day-to-day execution
- **Devs:** scheduled in-channel time, AMAs, dev diaries
- **PR:** press relationships, embargo coordination
- **External agencies:** creator network outreach, localization
